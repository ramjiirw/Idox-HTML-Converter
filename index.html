Help

Text to HTML — C# string output

Paste your email text into the input box — from any source (Outlook, Word, Notepad, etc). Press Enter for a new line, Enter twice to start a new paragraph.

Highlight text to use the floating toolbar: B bold, I italic, [ ] placeholder, • List bullet list, 1,2 List numbered list, 🔗 link.

For the link button: highlight display text and it asks for a URL; or highlight an existing URL to link it to itself.

Placeholders show as [word] in both the input and the output. The square brackets are a reminder that this part is a stand-in.

The buttons by the output: click Convert to produce the C# string. To start with, Convert is the only button you can use — once you've converted, the Output with [ ] / Output without [ ] switch and the Copy output button become available. Pick the bracket option you want, then click Copy output.

If you change the input text after converting, the output clears and the switch and copy button lock again — just click Convert once more to refresh the output.

When you mark something as a [placeholder] — like [Refval] — it can mean one of two things, and the Output with [ ] / Output without [ ] buttons let you choose which:

It's a true placeholder. The text inside the brackets is just a label reminding you that this part still needs work. After you paste the output into your C#, you'll replace both the brackets and the word inside with the correct string. For this, use Output with [ ] (the default) — the brackets stay in as a clear reminder of what's left to do.

It's already the exact string you want. The word inside the brackets is the final string. Only the brackets need to go. Use Output without [ ] — it strips the brackets for you (so [Refval] becomes Refval), saving you from deleting every set by hand.

The brackets always stay visible in the input above — these buttons only change what gets copied.

Insert table: click the ▦ Insert table button in the Input header to drop a table at your cursor. The table appears in the input as a small pill (e.g. ▦ Table 2×2) — hover to see a hint, click the pill for a popover showing the rendered preview with Edit / Delete / Close buttons. Backspace removes the pill like any character.

When you Convert, each pill becomes a self-contained table in the C# output at its position in the email.

Worked example:

You paste / type

Dear [ToName], We received your application today. Kind Regards LDC

You get

"<p>Dear " + [ToName] + ",</p>" + "<p>We received your application <strong>today</strong>.</p>" + "<p>Kind Regards<br>" + "LDC</p>"

Convert plain C# to HTML C#

Click Convert plain C# to HTML in the side panel to open a separate view. Paste your existing plain-text email body (the kind that uses Environment.NewLine) into the input box, and the converted HTML version appears automatically in the output. Edits to the input update the output live. Use this to upgrade old-style email templates to the new HTML format.

Highlight text in the input to B bold or I italic any section before conversion.

Important reminder: after converting, you also need to add this line to your C#:

mailMessage.IsBodyHtml = true;

Place it directly below the line mailMessage.Body = emailbody; in your code. While you're using the plain converter, a reminder box appears in the side panel with a Copy button for this line.

Worked example:

You paste

"Dear " + ToName + "," + Environment.NewLine + Environment.NewLine + "We received your application." + Environment.NewLine + Environment.NewLine + "Kind Regards";

You get

"<p>Dear " + ToName + ",</p>" + "<p>We received your application.</p>" + "<p>Kind Regards</p>"

HTML Tables

Click HTML Tables in the side panel to build a two-column (or more) HTML table for property statements and similar label/value layouts. Set the columns and rows with the +/− steppers, then fill in the cells.

Highlight text inside a cell to use the floating toolbar: B bold, I italic, [ ] placeholder, 🔗 link. Bullet and numbered lists are hidden inside table cells.

Right padding applies to every column except the last one (default 15px). Row spacing applies to every row except the last one (default 0px) — increase it for visual gap between rows in the rendered email.

Empty cells are skipped in the C# output entirely (no <td></td>). The preview shows exactly how the table will render in the email.

When you mark something as a [placeholder] — like [Refval] — it can mean one of two things, and the Output with [ ] / Output without [ ] buttons let you choose which:

It's a true placeholder. The text inside the brackets is just a label reminding you that this part still needs work. After you paste the output into your C#, you'll replace both the brackets and the word inside with the correct string. For this, use Output with [ ] (the default) — the brackets stay in as a clear reminder of what's left to do.

It's already the exact string you want. The word inside the brackets is the final string. Only the brackets need to go. Use Output without [ ] — it strips the brackets for you (so [Refval] becomes Refval), saving you from deleting every set by hand.

Click whichever you need, then Copy output. The brackets always stay visible in the cells and the preview above — these buttons only change what gets copied.

Use the × buttons above each column or beside each row to delete that column or row. If the cells being removed contain content, you'll be asked to confirm.

Worked example — a 2×2 table with two label/value rows, with the right column bolded and marked as placeholders:

You build

Case Reference: [Refval] Site Address: [SiteAddress]

You get

"<table>" + "<tr>" + "<td style='padding-right:15px;'>Case Reference:</td>" + "<td><strong>" + [Refval] + "</strong></td>" + "</tr>" + "<tr>" + "<td style='padding-right:15px;'>Site Address:</td>" + "<td><strong>" + [SiteAddress] + "</strong></td>" + "</tr>" + "</table>" +
